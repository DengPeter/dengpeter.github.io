git服务器搭建

1. 搭建简易服务器

通过ssh协议搭建简易的git服务器

在远程服务器上:

1. 创建git用户
  useradd git -c'git code control' /bin/git-shell

2. 创建git代码仓库目录
  mkdir /git #可用在任意地方创建任意目录名称作为git代码仓库使用
  chown git.  /git  #需要git用户有权限

3. 创建代码仓库(以blog项目为例)
  cd /git
  mkidr blog.git
  cd blog.git
  git init --bare

在开发人员机器上

1. 将ssh生成的公钥写入到远程服务器git用户的.ssh/authorized_keys中,注意authorized_kesy的权限是600
  cat ~/.ssh/id_rsa.pub

2. 将blog项目代码推送到远程仓库
  cd blog
  git init
  git add .
  git commit 'first commit blog project'
  git remote add origin git@hostname:/git/blog.git
  # 远程服务器地址 的几种形式 #默认是ssh协议
  # ssh://git@hostname:/git/blog.git
  # ssh://git@hostname:端口号/git/blog.git
  git push origin master

3. 这下就可用clone提交的代码了
  git clone git@hostname:/git/blog.git

git ssh服务器搭建非常的简单，但是只要能登录的用户对代码都有读写权限。一般公司都还是会根据不同的项目开发人员来设定权限的。我们通过gitosis来实现权限精细管理。

2. gitosis权限控制

安装gitosis

$ git clone https://github.com/tv42/gitosis.git
$ cd gitosis
$ yum install python-setuptools
$ sudo python setup.py install

默认 Gitosis 会把/home/git 作为存储所有 Git 仓库的根目录，这没什么不好，不过我们之前已经把项目仓库都放在 /git 里面了，所以为方便起见，我们可以做一个符号连接，直接划转过去，而不必重新配置：

$ ln -s /git /home/git/repositories

Gitosis 将会帮我们管理用户公钥，所以先把当前控制文件改名备份，以便稍后重新添加，准备好让 Gitosis 自动管理 authorized_keys 文件：

$ mv /home/git/.ssh/authorized_keys /home/git/.ssh/authorized_keys.bak

$ $ sudo -H -u git /opt/python2.7/bin/gitosis-init <.ssh/authorized_keys.bak
  Initialized empty Git repository in /coldata1/git/gitosis-admin.git/
  Reinitialized existing Git repository in /coldata1/git/gitosis-admin.git/

这样该公钥的拥有者就能修改用于配置 Gitosis 的那个特殊 Git 仓库了。接下来，需要手工对该仓库中的 post-update 脚本加上可执行权限：

  $ sudo chmod 755 /git/gitosis-admin.git/hooks/post-update

基本上就算是好了。如果设定过程没出什么差错，现在可以试一下用初始化 Gitosis 的公钥的拥有者身份 SSH 登录服务器，应该会看到类似下面这样：

  $ ssh git@gitserver
  PTY allocation request failed on channel 0
  ERROR:gitosis.serve.main:Need SSH_ORIGINAL_COMMAND in environment.
    Connection to gitserver closed.

说明 Gitosis 认出了该用户的身份，但由于没有运行任何 Git 命令，所以它切断了连接。那么，现在运行一个实际的 Git 命令 — 克隆 Gitosis 的控制仓库：

# 在你本地计算机上
$ git clone git@gitserver:gitosis-admin.git
$ git clone git@git.ztgt.cn:gitosis-admin.git

这会得到一个名为 gitosis-admin 的工作目录，主要由两部分组成：

➜  gitosis-admin git:(master) ls ./*
./gitosis.conf
./keydir:
afei@localhostmac.pub

gitosis.conf 文件是用来设置用户、仓库和权限的控制文件。keydir 目录则是保存所有具有访问权限用户公钥的地方— 每人一个。在 keydir 里的文件名（比如上面的 afei@localhostmac.pub）应该跟你的不一样 — Gitosis 会自动从使用 gitosis-init 脚本导入的公钥尾部的描述中获取该名字。

看一下 gitosis.conf 文件的内容，它应该只包含与刚刚克隆的 gitosis-admin 相关的信息：

➜  gitosis-admin git:(master) cat gitosis.conf
[gitosis]

[group gitosis-admin]
members = afei@localhostmac
writable = gitosis-admin


$ cat gitosis.conf
[gitosis]

[group gitosis-admin]
members = afei@localhostmac
writable = gitosis-admin

members 是用户名,取自keydir中的文件名afei@localhostmac.pub去掉.pub后缀；可以添加多个用户以空格分割 writable 是项目名称;项目名称的形式可以是writable = app/adroid.git writable = app/ios.git;这样在服务器上存放的时候。android和ios代码库就会放在app目录下面。

一个完整的例子：

    #在本机(管理员的本地代码库中修改)
    1. 添加新用户
      ➜  gitosis-admin git:(master) ls
      gitosis.conf  keydir/
      ➜  gitosis-admin git:(master) cp ~/lengchuan.pub keydir #将lengchuan.pub 的公钥复制到keydir目录中

    2. 添加新项目
      ➜  gitosis-admin git:(master) vim gitosis.conf
      [group app]
       members = lengchuan afei #这两个用户都能对adroid项目有读写权限
       writable = app/android.git #添加一个android.git项目到app目录下;不需要提前在服务器上面建裸库，第一次推送的时候会创建

   3. 将改动推送到远程仓库
      ➜  gitosis-admin git:(master) git push origin master


   #在用户lengchuan机器上

     ➜ cd ~/adroid
     ➜  adroid ls #项目中只有一个readme文件
     readme.md
     ➜  adroid git init #初始化仓库
     ➜  adroid git:(master) ✗ git add readme.md #将代码提交到本地仓库
     ➜  adroid git:(master) ✗ git commit -m'add readme.md' readme.md
     ➜  adroid git:(master) git remote add origin git@git.example.com:app/android.git #添加远程仓库地址
     ➜  adroid git:(master) git push origin master #将代码推送到远程仓库

接下来有权限的用户就可以克隆代码了。

git clone git@git.example.com:app/android.git

权限：

readonly writable
