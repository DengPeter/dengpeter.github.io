gitlab折腾记

2016-08-19
linux

文章目录

终于用上了gitlab感觉代码管理如此亲切。

写在前面，看到别人使用gitlab神似github，于是决定研究一下。按照惯例所有的服务都采用统一编译管理。 开始在centos上通过安装，安装成功后使用到git push的时候报错gitlab: git-http-push failed, return code 22官方issues 属于gitlab的bug官网上issues中并没有解决,尝试了各种解决方法决定放弃。由于官方推荐的是debin系，决定采用ubunt安装，结果还是同样的问题无解。 最后还是采用官方打包好的deb包run起来，尼玛可是数据库默认竟然是postgresql，换mysql结果又来一个无解的问题，还是乖乖的用postresql吧，这里记录的postgresql相关内容。 本篇记录下整个过程中遇到的坑。

简要记录下对gitlab架构的梳理：

gitlab: rails框架写成的web端

gitlab-shell: git动作的实现

gitlab-workhorse: 请求(任务)分发

编译安装官方指南

deb包安装官方指南

按照官方指南一步步操作：

  1. 安装需要的组建ssl,postfix(发送邮件服务端) sudo apt-get install curl openssh-server ca-certificates postfix

  2. 启用官方源 curl -sS https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.deb.sh | sudo bash sudo apt-get install gitlab-ce

  3. 配置并启动gitlab sudo gitlab-ctl reconfigure 上面3步困难的1.是官方源是国外的，下载安装包(280MB)比较慢,安装包大主要是把需要的rubygem包都已经打包在里面了， 2.更新到中间会assleep一下卡了我好长时间，后面发现是ruby源慢造成的。果断换https://ruby.taobao.com。 gem sources –add https://ruby.taobao.org/ –remove https://rubygems.org/

  4. 根据自己的实际情况配置： 修改访问地址: vim /etc/gitlab/gitlab.rb; #external_url ‘http://git.example.com' 重启重新初始化gitlab gitlab reconfigure 浏览器访问 你的url 输入密码就可以登录了

  5. 问题来了 deb安装完的相关工作路径在那里？ 配置路径: /etc/gitlab 配置/数据/日志存放路径: /var/opt/gitlab

web程序路径: /opt/gitlab

如何登录postgresql数据库？ 这个问题也是困了好久,在/var/opt/gitlab/gitalb-rails/etc/发现database.yml文件，看到host居然是/var/opt/gitlab/postgresql 果然用127.0.0.1 无法登录换做：sudo -u gitlab psql -h /var/opt/gitlab/postgresql -d databasename竟然登录成功了，不知道这是什么思路。

真搞不明白官方版本迭代如此快，为何跨平台遇到的问题，却迟迟不解决。 写完了发现有点乱，只记录了主要的东东。
