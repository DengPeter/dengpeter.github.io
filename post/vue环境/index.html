vue系列-vue开发环境搭建

2020年1月7日
| 前端
| 阅读

前端的技术发展飞速，封装了很多框架隐藏了很多细节，让功能实现越来越方便，但也越来越搞不明白底层隐藏的部分。用了一段时间的vue现在抽空来做一下整理和总结。期望可以比较系统和完整的梳理一遍，实际（纯属个人笔记完整和严谨性都有待提升）。

第一篇就先整理开发环境的部分，cnd引入的方式就忽略了直接进入脚手架的部分。

前置条件

  1. 安装nodejs 安装包地址 根据自己的系统平台下载对应的安装包进行安装。 安装完成后在命令行查看node和npm是否安装成功.

npm是nodejs的包管理器。方便我们从npm服务器上下载三方库。同样我们打包好的node程序也可以发布到npm上面。

下面是一些npm的常用命令:

  • npm install 安装
1
2

npm install sass-load --save
// 示例中安装了 sass-load包

install 的可选参数说明:

–save //简写 -S
–save-dev // 简写 -D
上面两个参数会把安装信息写入到当前目录下的package.json中
–save 保存到 dependencies,
–save-dev 保存到 devDependencies中

配置文件解释

  • package.json

    package.json是项目所需的运行和开发环境。 在nodejs中每个项目下都会有一个package.json文件它定义了这个项目的相关信息.比如: 项目所需的各种模块，项目的配置信息（名称、版本、许可等元数据）。 npm install命令根据这个配置文件来自动下载安装所需的模块。

下面介绍一下部分字段：

  • scripts

    scripts 指定了npm对项目执行命令的缩写.

1
2
3
4
5
6

"scripts": {
    "serve": "vue-cli-service serve",
    "build": "vue-cli-service build",
    "lint": "vue-cli-service lint",
    "test:unit": "vue-cli-service test:unit"
  }

  • dependencies和devDependencies

    dependencies指定了项目运行所依赖的模块，devDependencies指定项目开发所需要的模块。 它们都指向一个对象。该对象的各个成员，分别由模块名和对应的版本要求组成，表示依赖的模块及其版本范围。

1
2
3
4
5
6
7

 "dependencies": {
    "core-js": "^3.4.4",
    "vue": "^2.6.10",
    "vue-router": "^3.1.3",
    "vuex": "^3.1.2"
  }
  ...
