virtualenv

1. 安装virtualenv

    $ pip install virtualenv

2. 创建虚python虚拟环境

创建python虚拟环境，可以指定不同的版本，安装不同的第三方库. 下面安装django1.7版本为例

   $ virtualenv [虚拟环境名称]  #格式
   $ virtualenv django16env     #实例
   $ cd django16env/bin
   $ ./pip install jdango==1.7

3. 启动/退出虚拟环境

    $ cd django16env/bin/activate
    $ source activate   #启动环境  ＃ 命令前会多个括号，里面会显示虚拟环境名称
    $ deactivate        #退出环境

4. 使用virtualenvwrapper 管理虚拟环境

virtualenvwrapper 提供了对虚拟环境的方便快捷的管理方式：创建，删除，切换，查看等操作。并且将虚拟环境集中到一个地方使得对多版本多项目的虚拟环境的管理变得方便。

4.1 安装virtualenvwrapper

$ sudo  pip install virtualenvwrapper  #这是装在主系统中的
$ mkdir ~/virenv                       #创建管理目录
$ cat /etc/profile.d/virtualenv.sh     #配置环境变量
$    export WORKON_HOME='~/virenv'
     source /usr/bin/virtualenvwrapper.sh  #注意不同平台和不同版本的路径不一致,可以使用：type virtualenvwrapper.sh 来确定准确的位置

4.2 常用命令

    $ mkvirtualenv   [虚拟环境名] #创建虚拟环境,创建完后会自动切换环境

    $ lsvirtualenv -b             #查看已经创建的虚拟环境 

    $ workon [环境名称]           #切换虚拟环境 

    $ echo $VIRTUAL_ENV           #查看虚拟环境路径

    $ lssitepackages              #查看安装了哪些包

    $ cdvirtualenv                #进入当前环境目录

    $ cdvirtualenv [子目录]       #进入当前环境子目录

    $ cdsitepackages [子目录]     #进入cdsitepackages 及子目录

    $ toggleglobalsitepackages    #控制环境是否使用global site-packages：toggleglobalsitepackages

    $ cpvirtualenv [source] [dest] #复制虚拟环境

    $ deactivatea                 #退出虚拟环境

    $ rmvirtualevn [虚拟环境名]   #删除虚拟环境

5 错误归总

chrysan@chrsyan:~$ mkvirtualenv pelican
ERROR: virtualenvwrapper could not find virtualenv in your path   #是因为系统中的virtualenv 不能正常运行，重新安装即可
chrysan@chrsyan:~$ virtualenv pelican
The program 'virtualenv' is currently not installed. You can install it by typing:
sudo apt-get install virtualenv
chrysan@chrsyan:~$ sudo apt-get install virtualenv
