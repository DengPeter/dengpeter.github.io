编译subversion1.8

svn 突然连不上服务器，执行svn任何命令会卡很久，最后提示:”option Error 安全连接被切断” “在其他服务器上可以正常使用基本排除是服务器端的问题，再检查本地也没找出什么原因来。最后决定重新编译subversion，从1.6升级到1.8，升级完就可以使用了。困了几天，虽然不清楚为什么不能用，但现在总算可以使用了。记录下新版本的svn安装过程。

  • 软件包清单:

apr,apr-util,serf(替换老版本的neon)，scons（安装serf使用),subversion

  • 依赖清单:

sudo yum -y install expat-devel apr 和 apr-util 之前已经安装在/opt下面了，这里就略过。

** scons安装

  wget -c  http://superb-sea2.dl.sourceforge.net/project/scons/scons/2.3.3/scons-2.3.3.tar.gz

  tar axf scons-2.3.3.tar.gz
  sudo python setup.py install

** serf安装

  wget -c https://archive.apache.org/dist/serf/serf-1.3.8.tar.bz2
  或者 svn checkout https://svn.apache.org/repos/asf/serf/trunk serf-trunk
  tar axf serf-1.3.8.tar.bz2
  cd serf-1.3.8
  scons PREFIX=/opt/serf APR=/opt/apr APU=/opt/apr-util
  scons -Y ../serf-1.3.8
  scons PREFIX=/opt/serf LIBDIR=/usr/lib64
  sudo scons PREFIX=/opt/serf install

** subversion安装

   wget -c http://mirror.olnevhost.net/pub/apache/subversion/subversion-1.8.16.tar.gz

  tar axf subversion-1.8.16.tar.gz
  cd subversion-1.8.16
  wget http://www.sqlite.org/sqlite-amalgamation-3071300.zip
  unzip sqlite-amalgamation-3071300.zip
  mv sqlite-amalgamation-3071300 sqlite-amalgamation
  ./configure --prefix=/opt/svn --with-apr=/opt/apr --with-apr-util=/opt/apr-util/ --with-serf=/opt/serf/
  make
  sudo make install
  sudo mv  /usr/bin/svn{,.1.6}
  sudo ln -s /opt/svn/bin/svn /usr/bin/svn

以前遇到的错误： E170000: Unrecognized URL scheme for http，原因没有安装serf
